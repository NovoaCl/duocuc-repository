### Persistencia de datos

Persistencia significa guardar datos de forma permanente, sin perderse cuando la aplicacionse detiene o reinicia

- Se almacenan en sustemas externos, se conserva historico integridad y capacidad de relación.
- Multiples microservicios puedden acceder a la misma información(o sincronizarla).  

**Jakarta Persistence** (JPA) proporciona una forma coherente y simplificada de interactuar con bases de datos relacionales mediante el mapeo objeto-relacional(ORM).

- Simplifica la persistencia de datos
- Prortabilidad
- Reduccion de código boilerplate

**ORM**: Tecnica de programación. Actua como mediador entre una db relacional u un objeto de lenguaje de programación orientado a objeto.
- Mapear: Transformar una clase, objeto o atributo java en una tablaSQL, registro o columna.

**Hibernate:** Framework ORM más populares. Hace el mapeo

**Spring Data JPA**
- Reduce codigo repetitivo
- simplifica operacoines CRUD
- Permite consultas automaticas con delived queries
- Evita escribir SQL manual para lo basico
- Antaciones: 
        - findAll()
        - finById()
        - dave()
        - delete()
        - ecistById()
        - page/Sort()
